Installation

Two ways to run HomRec — a prebuilt executable that requires nothing extra, or from source if you want to dig in.

Option A — .exe (recommended)

FFmpeg is already bundled inside the archive — no extra downloads.

1. Download
Go to Releases and grab the latest .zip
2. Unzip
Extract anywhere — no installer, no admin rights required
3. Run
Launch !hr.exe
Some antiviruses (Kaspersky, Avast) may flag HomRec because it's a new program. It is not a virus — full source is on GitHub. Add the HomRec folder to exceptions if needed.

Option B — from source

# 1. Clone the repo
git clone https://github.com/homaaio/homrec.git
cd homrec

# 2. Install Python dependencies
pip install -r requirements.txt

# 3. Place ffmpeg.exe in the HomRec folder, or add to PATH
#    Download from https://ffmpeg.org/download.html

# 4. Run
python homrec.py

Hotkeys

Global hotkeys work even when the HomRec window is not focused. Custom bindings can be added via the developer console.

KeyAction
F9Start / Stop recording
F10Pause / Resume recording
F11Toggle fullscreen
Ctrl+Shift+TOpen / close developer console
EscClose developer console

To bind custom hotkeys use !connect --function in the console:

!connect --function !rec to ctrl+shift+r
!connect --function "!start --rec 1" to ctrl+F9 #name="start-rec"

Six Python packages. Two are optional.

PackagePurposeRequired
opencv-pythonVideo encoding and frame processingYes

Audio capture (microphone + desktop) is handled via native WASAPI C++ libraries — no extra audio packages needed.

Developer Console

Open with Ctrl+Shift+T. Close with Esc or the × button. Type !help for a quick command list without leaving the console.

The console gives you full programmatic control over HomRec — start and stop recording, create automation rules, bind hotkeys, run scripts, and inspect internal state.

Conventions

Named parameters, rule step syntax, and common patterns shared across all commands.

Named parameters

Parameters are passed as #key="value" or #key=value. Always wrap values that contain spaces in quotes.

!create --window #name="My Dashboard" #bg=black #size=(800x600)
ParameterTypeDescription
#name="..."stringObject name (window, rule, AE, alias…)
#ms=NintTime in milliseconds
#count=NintNumber of repetitions
#file="..."stringFile path
#val="..."stringGeneric value (context-dependent)

Rule step syntax

Steps inside a rule body are separated by ;. The keyword then is a harmless separator and is ignored during execution.

!create --rule #name="auto"; !start --rec 1 then; !echo --ok "done"

Global flags

These flags can be appended to any command.

FlagEffect
-s / --silentSuppress all console output for this command
-qSkip confirmation prompts
-return / -retPrint only 1 (success) or 0 (failure) instead of text
!connect --rule #name="logger" -s
!create --window #name="debug" -return

Math in arguments

The expression {int.random(a, b)} can be used anywhere in any command — it is replaced by a random integer in [a, b] before parsing.

!echo Random: {int.random(1, 100)}
!create --window #name="win_{int.random(1,999)}"
!delay #ms={int.random(500, 2000)} !rec

Recording

!rec

Toggle recording — starts if not active, stops if active.

!rec

!start --rec 1|0

Explicitly start (1) or stop (0) recording. Does nothing if already in the requested state.

!start --rec 1   # start recording
!start --rec 0   # stop recording

Windows

Windows are stored in create/windows.json. Each window has a name, style properties, and an enabled status.

!create --window

!create --window #name="..." [#bg=COLOR] [#fg=COLOR] [#size=(WxH)]
                 [-o] [-s] [-n] [-c] [-d]
                 [--topmost] [--borderless] [--resizable] [--minimized]
                 [--center] [--opacity #val=N] [--icon #path="..."]
                 [--title #val="..."]
Parameter / FlagDescription
#name="..."Required. Window name
#bg=COLORBackground color (CSS name or hex)
#fg=COLORText / foreground color
#size=(WxH)Window dimensions, e.g. (1024x768)
-oRegister only — do not open immediately
-cAuto-connect after creation
-dCreate as disconnected (window not opened)
--topmost / -tAlways on top
--borderless / -bBorderless window
--resizable / -rResizable window
--centerCenter on screen when opened
--opacity #val=NOpacity level (0–100)
!create --window #name="Dashboard" #bg=#1e1e2e #fg=#cdd6f4 #size=(800x500)
!create --window #name="debug" -o -d
!create --window #name="top" --topmost --center #size=(400x300)

!create --window --notepad

Creates a file inside ./create/ and opens it in the system editor.

!create --window --notepad #name="notes"
!create --window --notepad as .md #name="readme"
!create --window --notepad as .py #name="script"

!start --window / !edit --window

Re-opens a window previously registered via !create --window. Both commands are equivalent.

$rm --window

$rm --window #name="debug" -q
$rm --window #name="notes" --purge
$rm --all --window -y

Rules

Rules are sequences of commands stored in create/rules.json. When a rule is connected, its body executes immediately.

!create --rule

!create --rule #name="start"; !start --rec 1; !echo --ok "recording started"
!create --rule #name="once-only"; !echo "runs once" --once
!create --rule #name="loop3"; !echo "tick" --loop #count=3  -c
Flag / ParamDescription
#name="..."Required. Rule name
; step1; step2Rule body, steps separated by ;
-cExecute immediately after creation
-dSave as disconnected
--onceExecute only once
--loop #count=NRepeat steps N times
--delay #ms=NDelay between steps
--on-fail #cmd="..."Command to run on step failure

!connect --rule / !disconnect --rule

!connect    --rule #name="auto"
!disconnect --rule #name="auto"
!connect    --rule #name="auto" --toggle
!connect    --rule --all

!edit --rule

Replace the rule body. If no steps are provided, the current body is printed.

!edit --rule #name="auto"; !start --rec 1; !echo "updated"

AE Objects

AE (Anything Else) — extensible objects of arbitrary type, stored in create/ae.json. Currently the color type is supported.

!create --ae #type=color{rgb=(255,100,50)} #name="accent"
!create --ae #type=color{hex=(#FF6432)}    #name="orange"
!disconnect --ae #type=color #name="accent"

Connect & Disconnect

Enable or disable windows and rules. State is persisted in JSON registry files.

!connect --window

!connect --window #name="Dashboard" 1   # enable and open
!connect --window #name="Dashboard" 0   # disable
!connect --window #name="Dashboard" --toggle
!connect --window --all 1

Hotkey bindings

Bindings are stored in create/hotkeys.json and restored on startup.

!connect --function

!connect --function !rec to ctrl+shift+r
!connect --function "!start --rec 1" to ctrl+F9 #name="start-rec"
!disconnect #name="start-rec"    # unbind by alias

History & Aliases

!history

!history #count=50
!history --search "create"
!history --clear

The input field also supports ↑ / ↓ keyboard navigation through history.

!alias

Aliases are persisted in create/aliases.json. Once created, type the alias name as a command directly.

!alias #name="sr"  #cmd="!start --rec 1"
!alias #name="sp"  #cmd="!start --rec 0"
!alias --list
!alias --remove #name="sr"
sr     # executes !start --rec 1

Command execution

!repeat

!repeat #count=5 !echo "hello"

!delay

Non-blocking. The command runs in a background thread; the console remains interactive.

!delay #ms=3000 !rec
!delay #ms=500  !echo --ok "done"

!batch

!batch !start --rec 1 && !echo --ok "started" && !timer #name=t1 #ms=5000 !start --rec 0
!batch !ls && !status -x

!run

Execute a .hrc script file line by line. Lines starting with # are comments.

!run #file="setup.hrc"
!run #file="scripts/init.hrc" --echo-each --encoding cp1251

Timers & Watchers

!timer — one-shot

!timer #name="stop-rec" #ms=10000 !start --rec 0
!timer --cancel #name="stop-rec"
!timer --list

!watch — periodic

!watch #name="ping-loop" #ms=5000 !ping
!watch #name="screenshot" #ms=60000 !echo "tick" --max-runs #count=10
!watch --stop #name="ping-loop"
FlagDescription
#ms=NInterval between runs in ms
--max-runs #count=NAuto-stop after N executions
--jitter #ms=NRandom deviation of interval ±N ms
--stop #nameStop a named watcher
--listList all active watchers

System commands

!ping
Check DLL ↔ Python bridge. Prints pong and round-trip time in ms.

Print console version (C++ DLL), Python bridge version, and interpreter version.
!exit
Force-quit the application. Stops recording, kills ffmpeg, removes tray icon.
!clear
Clear the console output area.
!help [-w]
Print command list. Without -w also opens the full documentation in the browser.

Deleting objects

$rm

$rm --window #name="debug" -q
$rm --rule   #name="auto"
$rm --ae     #name="accent" -q
$rm --all --window -y
$rm --all --rule --if-disconnected -y
$rm --window #name="notes" --purge -q
FlagDescription
-q / -yNo confirmation prompt
--purgeAlso delete the physical file and linked hotkeys
--if-disconnectedDelete only if currently disabled
--allDelete all objects of the specified type

Architecture: DLL ↔ Python

The console is split into two layers communicating via a callback bridge.

hr_console.dll — Win32 / GDI (C++)
User types a command. Built-in dispatcher handles:
!help !rec !open !exit !date !homrec
Everything else → forward_to_python(raw) via CB_COMMAND callback
↓ CB_COMMAND callback
hr_console_bridge.py (Python)
NativeConsole._dispatch_extended handles all extended commands:
!create !connect !disconnect $rm !ls
!alias !timer !watch !batch and more

Public API (DLL)

FunctionDescription
hr_con_init(...)Initialise: register callbacks and paths
hr_con_set_command_cb(cb)Register the Python handler for extended commands
hr_con_toggle()Show / hide the console window
hr_con_set_recording(int)Sync recording state with the UI
hr_con_log_connected()Returns 1 if log writing is active, 0 if paused

Storage files

All registry state is persisted as JSON files in the create/ directory.

create/windows.json
Window registry — WindowRegistry
create/rules.json
Rule registry — RuleRegistry
create/ae.json
AE objects — AERegistry
create/aliases.json
Command aliases — AliasRegistry
create/hotkeys.json
Hotkey bindings — HotkeyManager

Examples

Auto-stop after 10 minutes

!create --rule #name="rec-10min"
  ; !start --rec 1
  ; !timer #name="stop" #ms=600000 !start --rec 0  -c

Workspace setup script

# workspace.hrc
!create --window #name="main" #bg=#1e1e2e #fg=#cdd6f4 #size=(1024x768)
!create --window --notepad as .md #name="notes"
!alias #name="rec"  #cmd="!start --rec 1"
!alias #name="stop" #cmd="!start --rec 0"
!connect --function !rec  to ctrl+F9  #name="start recording"
!connect --function !stop to ctrl+F10 #name="stop recording"
!echo --ok "workspace ready"

Run with: !run #file="workspace.hrc" --echo-each

Periodic status monitoring

!watch #name="monitor" #ms=30000 !status

Bind a toggle shortcut

!alias #name="tr" #cmd="!rec"
!connect --function !tr to ctrl+shift+space #name="toggle-rec"

Now Ctrl+Shift+Space starts or stops recording from anywhere.
