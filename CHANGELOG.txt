[Version 1.4.0] - 2026-04-30

=== Added ===

Help menu in the top menu bar with two options:
- Check for Updates — checks GitHub and shows result in a dialog
- Report Issue — opens GitHub Issues page in the browser

Desktop audio (system sound) capture via WASAPI loopback
(requires pyaudio-wasapi — see README for setup details)

Option to disable "minimize to tray on close" in Settings menu

Codec selector in Settings → Video tab — choose between:
- libx264 (CPU, H.264, universal default)
- libx265 (CPU, H.265, smaller files)
- h264_nvenc / hevc_nvenc (Nvidia GPU)
- h264_amf / hevc_amf (AMD GPU)
- h264_qsv / hevc_qsv (Intel GPU)

Hardware acceleration selector — auto-detects GPU decoder
(cuda for Nvidia, dxva2 for Intel, d3d11va for AMD)

Advanced Settings window (Settings → Preferences → Advanced Settings)
— separate window aimed at power users with full customization:

  Video: codec, bitrate, CRF, preset, pixel format, hardware acceleration
  Audio: sample rate, AAC bitrate, channel count
  Interface: theme (dark/light/custom with color picker), UI scale, font
  Recording: replay buffer duration, auto-stop by time or file size,
             output folder, file naming template

Import / Export profile as .hrc file — save and share your full
configuration in a single file

=== Changed ===

PC Analytics is now a single combined window with CPU, RAM and Disk
sections and a Refresh button — instead of three separate dialogs

Start/Stop buttons merged into one button that changes text and color
when recording starts. Pause and Stop are now compact and sit side by side

System audio capture chain now tries three methods in order:
1. WASAPI loopback with as_loopback=True (requires pyaudio-wasapi)
2. PyAudio direct open of Stereo Mix device (works with standard pyaudio)
3. ffmpeg dshow as last resort

Version bumped to 1.4.0

=== Fixed ===

FFmpeg console window no longer appears in front of the app after stopping
a long recording

Desktop audio not recording — _find_wasapi_loopback was implemented but
never called; dshow fallback only worked if Stereo Mix was enabled in
Windows, which is off by default on most modern PCs

Stereo Mix not detected — PyAudio on Windows returns device names as
UTF-8 bytes decoded as cp1251, causing Cyrillic names like "Стерео микшер"
to appear garbled. Fixed by re-encoding: name.encode('cp1251').decode('utf-8')

System audio not saved to final file — ffmpeg dshow process was not
receiving a proper stop signal (missing newline after 'q'), causing the
WAV file to be left unclosed and skipped during merge

WAV header written with wrong channel count when Stereo Mix device is
mono — channel count is now detected at open time and stored correctly

Fallback to first dshow device removed — previously if Stereo Mix was
not found the code silently fell back to devices[0] which was usually
the microphone, causing mic audio to be recorded twice

check_updates key missing from English language dictionary — caused
KeyError on startup when language was set to English

Volume multiplier condition was vol != 2.0 instead of vol != 1.0 —
volume scaling was always applied even at 100%

stop_audio_recording returned None instead of audio file path in
mic-only mode — caused merge_audio_video to be skipped entirely,
resulting in video with no audio track

merge_audio_video silently swallowed all exceptions — now logs ffmpeg
stderr so failures are visible in homrec.log
