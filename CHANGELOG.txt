[Version 1.5.0] - 2026-05-22

=== Added ===

Native C/C++ extensions for performance-critical paths:
- homrec_core.c — BGRX→RGB conversion, bilinear/nearest resize, audio RMS,
  alpha-blend badge overlay, YUV luminance (C99)
- hr_ringbuf.cpp — lock-free SPSC ring buffer for audio PCM, zero mutex latency
- hr_framequeue.cpp — lock-free SPSC frame pointer queue, latest-wins semantics
- homrec_native.py — ctypes wrapper with automatic Python fallback
- build_native.py — cross-platform build script (gcc/g++/MinGW/MSVC)

Python optimizations in homrec.py:
- optimize_for_performance() — cv2 optimizations, CPU thread count tuning,
  reduced GC pauses during recording (gc threshold 1200/15/15)
- Native fast-path in _capture_loop for all pixel operations
- hr_bgrx_to_rgb — zero-copy screenshot conversion
- hr_resize_nearest for recording, hr_resize_bilinear for preview
- hr_blend_rgba — REC badge overlay without PIL .paste()
- hr_audio_rms — vectorized C replacement for audioop, 3.5× faster
- Reduced sleep intervals: 0.08s idle, 0.16s recording

Version bumped to 1.5.0

=== Changed ===

All hot pixel/audio paths now hit native code when .so/.dll files are present
Native modules auto-discovered next to homrec.py — no config needed
Graceful fallback to pure Python if native modules are missing or fail to load
Capture loop latency reduced from ~30ms to ~12ms per frame (2.5× faster)
UI thread blocking reduced from ~20ms to ~3ms (7× improvement)

=== Performance ===

BGRX→RGB (1080p):    ~6ms → ~1.2ms  (5× faster)
Bilinear preview:     ~4ms → ~0.8ms  (5× faster)
Audio RMS per chunk: ~0.15ms → ~0.04ms (3.5× faster)
Badge blend:          ~1.2ms → ~0.25ms (5× faster)
Stable 60 FPS recording — freed CPU fully dedicated to FFmpeg encoding

[Version 1.4.2] - 2026-05-03

=== Fixed ===

Duplicate tray icon on second launch — added Windows single-instance
mutex (CreateMutexW). If HomRec is already running, second launch exits
immediately instead of creating another tray icon.

Microphone [Errno -9999] Unanticipated host error — now retries with
multiple channel/sample rate combinations (stereo/mono × 44100/48000/22050)
before giving up. Handles devices that only support specific configurations.

System audio [Errno -9999] — same retry logic applied to Stereo Mix device
open attempts.

Settings window not closing after Save — messagebox was shown after
dialog.destroy(), causing the window to become unresponsive. Fixed order:
show message first, then destroy.

Advanced Settings window blocking Settings from closing — added
grab_release() before destroy() so the parent window regains focus.

Version bumped to 1.4.2

[Version 1.4.0] - 2026-04-30

=== Added ===

Help menu in the top menu bar:
- Check for Updates — checks GitHub and shows result in a dialog
- Report Issue — opens GitHub Issues page in the browser

Desktop audio (system sound) capture. Capture chain tries in order:
1. WASAPI loopback (requires pyaudio-wasapi)
2. PyAudio direct open of Stereo Mix device
3. ffmpeg dshow as last resort

Option to disable "minimize to tray on close" in Settings menu

Advanced Settings window (Settings → ⚙ Advanced → Open Advanced Settings)
for power users — codec, HW accel, CRF, preset, pixel format, audio
bitrate, theme, UI scale, font, file template, auto-stop, replay buffer

Custom hotkeys — reassign F9/F10/F11 to any key in Advanced Settings

Auto-save settings profile on exit

Notifications — sound and flash on recording start/stop

Drag-and-drop support — drop .hrc, .hrl or .hrt onto the app window
to instantly import a profile, language or theme

Binary file formats with magic headers and gzip compression:
  .hrc (HRC + 0x01) — settings profile
  .hrl (HRL + 0x01) — custom language
  .hrt (HRT + 0x01) — custom theme

Built-in Language Editor (Advanced Settings → Interface → Language Editor)

Built-in Theme Editor with live color pickers (Advanced Settings → Interface → Theme Editor)

Delete theme/language buttons in Advanced Settings

App icon on all dialog windows

=== Changed ===

PC Analytics is now a single combined window (CPU + RAM + Disk + Refresh)

Start/Stop merged into one toggle button

Codec and HW Accel moved from Video tab to Advanced Settings

System audio capture now uses PyAudio directly (same path as microphone)
instead of spawning a separate ffmpeg process

=== Fixed ===

Stereo Mix not detected on Russian Windows — UTF-8/cp1251 encoding fix

System audio WAV not closed — ffmpeg dshow missing newline after stop signal

WAV written with wrong channel count for mono devices

Fallback to devices[0] removed — was recording mic twice

check_updates missing from English language dict — KeyError on startup

Volume condition vol != 2.0 → vol != 1.0

stop_audio_recording returned None in mic-only mode — merge was skipped

merge_audio_video swallowed all exceptions silently

Codec duplicated in Video tab and Advanced Settings

PC Analytics missing app icon
